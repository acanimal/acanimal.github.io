In the past, I have been using Dojo Toolkit for relatively little projects. I choose Dojo by its well designed API and its rich and homogeneous widget set. jQuery and jQueryUI are great projects, but I found the problem of consistency with jQuery plugins. Each one follow its own rules, each one has its documentation and API, etc. Because this, once I know Dojo I chose it as my preferred framework for web RIA applications.

When you start working with Dojo, you found there are two ways to make the things: programmatically or declaratively. The first the the "obvious" way, which is used by other projects like ExtJS, where you create new panels, buttons, grids, etc by code. On the other side, the Dojo's declarative way consist on HTML code where elements are tagged with some extra attributes. Later, when the page is loaded the Dojo toolkit is responsible to parse the DOM structure and transform tagged elements to Dojo widgets, for example, convert a HTML button into a Dojo button with a nice look and feel and improved events.
